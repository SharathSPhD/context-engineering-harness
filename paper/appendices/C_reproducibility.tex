\subsection{C.3 Random-seed contract}\label{c.3-random-seed-contract}

Every experiment fixes seeds in three places:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  The Python \texttt{random} module: \texttt{random.seed(seed)}.
\item
  NumPy: \texttt{np.random.default\_rng(seed)}.
\item
  Anthropic API: an explicit \texttt{seed} keyword on every \texttt{Messages.create} call (when supported by the model).
\end{enumerate}

Seeds used are \texttt{{[}1,\ 2,\ 3{]}} for the canonical L1/L3 runs, and a single \texttt{seed=0} for the deterministic L2 case study.

\subsection{C.5 Cost ledger}\label{c.5-cost-ledger}

The custom \texttt{CLIBudgetScheduler} writes \texttt{cost\_ledger.db} with one row per outgoing API call. To re-derive total token usage and total wall-clock spend:

\begin{verbatim}
uv run python -m harness.scheduler.report --db cost_ledger.db
\end{verbatim}

The ledger from our run is shipped as \texttt{experiments/results/cost\_ledger.snapshot.db} for inspection.

\subsection{C.6 Determinism audit}\label{c.6-determinism-audit}

\texttt{tests/test\_v2/test\_p7\_aggregate.py::test\_aggregate\_is\_deterministic} asserts that two consecutive runs of the aggregator produce byte-identical figure binaries and table strings. This pins the entire pipeline against silent stochastic drift.

\subsection{\texorpdfstring{C.7 What is \emph{not} deterministic}{C.7 What is not deterministic}}\label{c.7-what-is-not-deterministic}

The L1 and L3 layers depend on the Anthropic API. Anthropic does not guarantee bitwise-identical completions across provisioning windows even with a fixed \texttt{seed}. We therefore report multi-seed means with bootstrap CIs and paired permutation tests, not point estimates. The deltas reported in Sections 8 and 10 are robust to single-seed drift on the order of \(\pm 0.02\) on every metric we tested.

The L2 layer is fully deterministic: it is LLM-free.
