\section{Full MCP Tool Specifications}\label{full-mcp-tool-specifications}

This appendix gives the full input / output schema and behavioural contract for each of the 15 MCP tools shipped by \passthrough{\lstinline!pratyaksha-context-eng-harness!} v1.0.0. The canonical source is \passthrough{\lstinline!plugin/pratyaksha-context-eng-harness/mcp/server.py!}.

All tools share three host-platform invariants:

\begin{itemize}
\tightlist
\item
  \textbf{Audit hook.} Every successful tool call appends a single JSON-line to \passthrough{\lstinline!\~/.cache/pratyaksha/audit.jsonl!} (XDG-style cache path; survives plugin reinstall) via the \passthrough{\lstinline!\_audit(event, payload)!} helper.
\item
  \textbf{Budget hook.} Every tool call is preceded by the \passthrough{\lstinline!pretooluse-budget.sh!} lifecycle hook, which consults \passthrough{\lstinline!budget\_status!} and \textbf{logs} budget pressure by default (\textbf{advisory}). Optional strict refusal is available when \passthrough{\lstinline!PRATYAKSHA\_BUDGET\_STRICT=1!} and the hook is configured to deny over-threshold calls.
\item
  \textbf{Schema-strict I/O.} Every tool's input is validated by a Pydantic v2 model; every output is JSON-serialisable.
\end{itemize}

\subsection{\texorpdfstring{\texttt{context\_insert} --- avacchedaka insertion}{context\_insert --- avacchedaka insertion}}\label{context_insert-avacchedaka-insertion}

\begin{lstlisting}[language=Python]
class InsertInput(BaseModel):
    qualificand: str

    condition: str = ""

\textbf{Behaviour.} Generates a new \passthrough{\lstinline!id!} (UUIDv4), writes the row to the in-memory \passthrough{\lstinline!ContextStore!}, returns the serialised row.

\subsection{\texorpdfstring{\texttt{sublate\_with\_evidence} --- evidence-anchored sublation}{sublate\_with\_evidence --- evidence-anchored sublation}}\label{sublate_with_evidence-evidence-anchored-sublation}

\textbf{Signature.} \passthrough{\lstinline!sublate\_with\_evidence(args: SublateWithEvidenceInput) -> \{target: ContextElement, by: ContextElement\}!}

\begin{lstlisting}[language=Python]
class SublateWithEvidenceInput(BaseModel):
    target_id: str
    by_id: str
    reason: str
\end{lstlisting}

\textbf{Behaviour.} This is the \emph{primary} bādha primitive (Section 4.3). Sets \passthrough{\lstinline!target.status = "bādhita"!} and \passthrough{\lstinline!target.superseded\_by\_id = by\_id!}, retains the original for audit. Triggers the rule engine's \emph{dominance check} (Section 4.3) automatically against any other live items sharing \passthrough{\lstinline!(qualificand, condition)!} whose precision is dominated by \passthrough{\lstinline!by!}.

\subsection{\texorpdfstring{\texttt{detect\_conflict} --- Bayesian-margin conflict check}{detect\_conflict --- Bayesian-margin conflict check}}\label{detect_conflict-bayesian-margin-conflict-check}

\textbf{Signature.} \passthrough{\lstinline!detect\_conflict(args: RetrieveInput) -> \{conflicted: bool, margin: float, items: list[ContextElement]\}!}

\textbf{Behaviour.} Runs the Bayesian Beta-Bernoulli aggregator (Section 5.3) over the matching items. Returns \passthrough{\lstinline!conflicted=True!} when the posterior margin \(|\bar{H} - 0.5|\) falls below the threshold \(\tau\) (default 0.10).

\textbf{Signature.} \passthrough{\lstinline!list\_qualificands() -> \{qualificands: list[str]\}!}

\textbf{Behaviour.} Returns the deduplicated list of \passthrough{\lstinline!qualificand!}s across the live store. Used by Manas to map ``what topics do I currently know about?''.

class ClassifyInput(BaseModel):

\subsection{\texorpdfstring{Negative claims --- what the plugin does \emph{not} do}{Negative claims --- what the plugin does not do}}\label{negative-claims-what-the-plugin-does-not-do}

We assert and audit:

\begin{itemize}
\tightlist
\item
  \textbf{No reference to \passthrough{\lstinline!attractor-flow!}, \passthrough{\lstinline!ralph-loop!}, or \passthrough{\lstinline!triz-engine!}} anywhere in the shipped plugin tree.
\item
  \textbf{No hard dependency on \passthrough{\lstinline!vllm!}, \passthrough{\lstinline!mlflow!}, \passthrough{\lstinline!chromadb!}, or \passthrough{\lstinline!huggingface-hub!}.} The runtime requirements are \passthrough{\lstinline!mcp!}, \passthrough{\lstinline!pydantic!}, \passthrough{\lstinline!tiktoken!}, \passthrough{\lstinline!numpy!}, \passthrough{\lstinline!anthropic!}. The optional \passthrough{\lstinline!vllm!} / \passthrough{\lstinline!transformers!} path for the Qwen3 surprise backend (Section 5.6) is loaded only when the user opts in.
\item
  \textbf{No model fine-tuning anywhere.} Every ``agent'' in this work is a system-prompted role over an unmodified frontier LLM.
\end{itemize}

The audit reproducer is \passthrough{\lstinline!docs/no\_dev\_deps\_audit.md!} in the repository.
